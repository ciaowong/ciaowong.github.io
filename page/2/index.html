线程占用无法释放

发表于 2021-04-25 更新于 2021-04-26 分类于 项目

现象

平时3分过几秒钟完成的任务，生产上隔了几分钟才完成

推导

通过jstack 查询堆栈信息，相关脚本

大量线程处于 RUNNABLE 状态，定位到具体方法，是这个

statMktTradeHhr statMktTradePersonal mergeMktTrade

这个sql一直读区数据，导致线程没有回收，影响了后面其他任务。

正常的线程

解决方案

1.通过控制线程的超时时间，执行一段时间报错回收，不让其一直执行；

2.优化慢查询的sql

Refresh方法

相当于重启，销毁所有的 Bean 对象，重新生成

Bean的生命周期

  • 从循环依赖引出 Bean 创建的生命周期
  • Spring 通过3级缓存解决循环依赖问题
    • 2级缓存解决不了 AOP 问题（ 赋值给B的A是原始对象而不是代理对象）
  • 如果出现多个循环依赖，会创建多个 A 的代理对象，用二级缓存，存不完整的代理对象
    • 不完整的 Bean 不能放入单例池
    • 单例池存放的是有完整生命周期的 bean
  • 构造方法有相互依赖 Spring 解决不了，报错（通过 @Lazy 解决）

电商超卖问题

发表于 2021-04-07 更新于 2021-04-26 分类于 JAVA技术

怎么防止电商超卖？

情况1: 单机，高并发

用 synchronized 关键字加锁；

情况2: 多机

引入中介防止订单重复：

中介1:

生成订单通过 MySQL 主键，防止重复；

中介2:

通过 Redis setnx key value；引发下列问题：

Q&A

Q1: 无法释放锁？

A：加超时时间 expire time ;
Q2: 生成完成后其他请求获取不到锁？

A：要释放锁 delete key;
Q3: 怎么确定你删除的锁是你自己加的锁？

A：给锁增加线程唯一标识，删除前先判断标识是否匹配；
Q4: 超时时间内无法执行完怎么办？

A：通过 watch dog 临近结束前增加超时时间;(Redission)；
Q5: Redis 挂了，由于数据同步异步问题，导致2个线程获得同一把锁怎么办？

A：不用主从，用红锁( redis lock )部署奇数台独立的 Redis，依次获取锁，当成功获取半数以上的锁才视为加锁成功；
Q6: 2个线程都在争锁，其中一台 Redis 挂了，一个请求只要获得4台中的2台就拿到锁了；这时候重启，另一请求拿到了剩下3台的锁，也拿到锁成功？

A：告诉运维，等key都过期了再重启；
Q7: 如果多台 Redis 挂了怎么办？

A：部署多台 Redis 的目的就是为了防止某台挂掉，最后还挂了多台，要把运维开了。

终极解决方案 MySQL 乐观锁 + zk
用 zk 的临时节点当作版本号，存在数据库里。

大多数公司就用一台 redis 就够了
